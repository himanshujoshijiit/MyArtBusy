Quick Start — Run in 5 Minutes

Prerequisite: Install Docker Desktop and make sure it is running before you start.
1

Clone the repository

git clone https://github.com/himanshujoshijiit/MyArtBusy.git
cd MyArtBusy
2

Create environment file

# Windows
copy .env.example .env

# Mac / Linux
cp .env.example .env

Edit .env only when you need live Razorpay or WhatsApp. Demo mode works without keys.

3

Start all services

docker compose up --build

First build takes 3–5 minutes. Wait until you see all containers healthy.

Windows shortcut: double-click start.bat

# Run in background (detached)
docker compose up --build -d
4

Open the app

URLWhat it is
http://localhost:3000Main app — start here
http://localhost:8080/actuator/healthJava API health check
http://localhost:8000/healthPython search API health
http://localhost:8000/docsFastAPI Swagger documentation
Open MakeupSeven App →
5

Stop the app

docker compose down

# Stop and remove database volume (fresh start)
docker compose down -v

Features

Core 1 — MUA Discovery Engine

🔍

Occasion-based search

Search by city + occasion — e.g. "Bridal Bengaluru", "Party Indiranagar". Portfolio drives discovery, not price.

🖼️

Artist profiles

Portfolio gallery, services, pricing, verified reviews, skin tone expertise, response time label.

🎛️

Advanced filters

Skin tone, min/max budget, min rating, available date, top artists only, locality, sort by rating/price/bookings.

Core 2 — Booking & Payment

📅

Calendar availability

MUAs add slots from dashboard. Clients pick date & time when booking. Public availability on artist profiles.

💳

Razorpay deposit (25%)

Secure upfront deposit. Mock mode works without API keys for demo. Live keys from Razorpay dashboard.

📱

WhatsApp confirmation

Auto message to client & artist after deposit payment. Mock mode logs to console without Meta API keys.

✍️

Contract e-sign

Digital service agreement generated after payment. Client signs at /contracts/[bookingId].

Core 3 — Reviews & Trust

✅

Verified reviews only

Only clients who completed a booking can leave a review. WhatsApp sends review link after service.

👑

Top Artist badges

Auto-awarded at 4.5+ rating with 5+ reviews. Featured/sponsored artists appear with badge in search.

Core 4 — MUA Business Dashboard

📊

Earnings tracker

Gross & net earnings (after 10% commission). Monthly and total stats.

🗓️

Booking calendar

Manage availability slots, view upcoming bookings, confirm/complete/cancel.

👤

Client face profiles

Skin tone, allergies, notes, past looks. Auto-created when booking is completed.

💄

Kit inventory

Track products, expiry dates, low-stock alerts on dashboard.

⚙️

Profile editor

Update bio, pricing, add portfolio images & services from dashboard.

⭐

Pro subscription

₹999/month — unlimited bookings, full analytics. Free tier: 3 bookings/month.

Core 5 — Education & Academy (Phase 2)

🎓

Online courses

Course catalog at /courses. 70% revenue to instructor, 30% to platform. 3 demo courses seeded (Bridal Masterclass, Editorial, Beginner Basics).

Production Deployment

Option A — Single VPS (DigitalOcean / AWS EC2 / Hetzner)

# On your server
git clone https://github.com/himanshujoshijiit/MyArtBusy.git
cd MyArtBusy
cp .env.example .env
# Edit .env — set JWT_SECRET, Razorpay keys, FRONTEND_URL, API URLs

docker compose up -d --build

Put nginx or Caddy in front for HTTPS and map your domain to ports 3000, 8080, 8000.

Option B — Cloud (scale globally)

ComponentSuggested platform
FrontendVercel, AWS Amplify, or Docker on ECS
Java APIAWS ECS, Google Cloud Run, Railway

DatabaseAWS RDS, Supabase, or Neon PostgreSQL

Local Development (Without Full Docker)

Requires: Java 17+, Maven, Python 3.12+, Node.js 20+, PostgreSQL

# Terminal 1 — Database only via Docker
docker compose up postgres -d

# Terminal 2 — Java API
cd backend-java
mvn spring-boot:run

# Terminal 3 — Python search API
cd backend-python
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000

# Terminal 4 — Frontend
cd frontend
npm install
npm run dev

Frontend dev server: http://localhost:3000 (with hot reload)

Troubleshooting

ProblemSolution
Docker not startingEnsure Docker Desktop is running. Wait 30–60s after opening it.
Port already in useStop other apps on 3000/8080/8000/5432 or change ports in docker-compose.yml
Frontend can't reach APICheck NEXT_PUBLIC_API_URL and NEXT_PUBLIC_SEARCH_URL in .env
Java API not healthyWait 60s for startup. Run docker compose logs backend-java
No artists showingDatabase may be empty. Run docker compose down -v then docker compose up --build to re-seed
Courses not showingRun docker compose restart backend-java to trigger course seeding
Payment failsWithout Razorpay keys, mock mode auto-confirms. Add test keys from Razorpay dashboard for real flow.
View logsdocker compose logs -f or docker compose logs backend-java -f
Rebuild after code changesdocker compose up --build -d

Useful commands

# Check running containers
docker compose ps

# Restart a single service
docker compose restart backend-java

# Stop everything
docker compose down

# Fresh database (deletes all data)
docker compose down -v && docker compose up --build -d
